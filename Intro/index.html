1. Introduction to Computing

What is a Computer Language?

A computer language is a formal language used to give instructions to a computer so it can perform specific tasks.

Computers do not understand human languages like English or Tamil.

They only understand binary (0s and 1s).

So, we use programming languages as an intermediate layer.

Types of Computer Languages

1. Machine Language

  • Written in binary (0 and 1)
  • Directly understood by the CPU
  • No translation required
  • Example: 1010 0011

Key Point: Fast but very hard for humans to write and debug.

2. Assembly Language

  • Uses mnemonic codes instead of binary
  • Example: ADD A, B
  • Requires an assembler to convert into machine code

Key Point: Easier than machine language, but still low-level.

3. High-Level Language

  • Human-readable and easy to write
  • Examples: C, Java, Python
  • Requires a compiler or interpreter

Key Point: Most commonly used in real-world programming.

Why Do We Need Computer Languages?

  • Humans think in natural language
  • Computers understand only binary
  • So, a translator is required

Creating and Running a Program

Steps

  1. Write Code

    You write the program using a high-level language (C, Java, etc.).

  1. Compile

    The compiler converts your code into machine-level instructions.

  1. Link

    Required libraries (like stdio.h) are connected to your program.

  1. Execute

    The final program runs and produces output.

Example Program (C)

#include<stdio.h>

int main()
{
    printf("Hello World");
    return 0;
}

How It Actually Works

Think of it like watching a foreign movie:

  1. Writing the Code

    You write your program in C language and save it as hello.c.

    At this stage, it's just human-readable source code.

  1. Preprocessing

    This is the first hidden step done automatically.

    • It expands #include <stdio.h> by inserting the actual library code.
    • It removes all comments.
    • It processes macros (#define).
      The output is still C code, but fully expanded.
  1. Compilation

    The compiler now translates the preprocessed C code into assembly language.

    • Input: Expanded C code
    • Output: hello.s (assembly file)

    Assembly is low-level but still human-readable. It contains instructions specific to the processor.

  1. Assembly

    The assembler converts the assembly code into machine code (binary).

    • Input: hello.s
    • Output: hello.o (object file on Linux/Mac) or hello.obj (on Windows)

    This file contains actual binary instructions, but it's not yet a complete program.

  1. Linking

    The linker combines your object file with necessary library functions (like printf from the standard library).

    • Input: hello.o + library files
    • Output: a.out (default on Linux) or hello.exe (on Windows)

    This final file is a complete executable.

  1. Execution

    When you run the executable, the operating system loads it into memory, and the CPU starts executing the machine instructions.

    Your program finally runs and produces output: Hello World

2. Overview of C

The C programming language is one of the most important and influential programming languages in the history of computing.

It was developed by Dennis Ritchie in the year 1972 at Bell Laboratories (Bell Labs) in the United States. The main purpose of creating C was to develop the Unix operating system, which was one of the earliest and most powerful operating systems at that time.

Why C Was Created

Before C, programming was mainly done using:

  • Machine language (very difficult)
  • Assembly language (slightly better but still complex)

These languages were:

  • Hard to write
  • Hard to understand
  • Not portable (could not run on different machines)

So, there was a need for a language that was:

  • Easier to write
  • Efficient like low-level languages
  • Portable across systems

That’s where C came into the picture.

Evolution of C

C did not appear suddenly. It evolved from earlier languages:

  • BCPL (Basic Combined Programming Language)
  • Then → B language (developed by Ken Thompson)
  • Finally → C language (improved by Dennis Ritchie)

C was designed by improving the limitations of B language and adding more powerful features.

Role of C in Unix Development

C became very popular because it was used to develop the Unix operating system.

Before this:

  • Operating systems were written in assembly language

But Unix was written in C, which made it:

  • Easier to maintain
  • Easier to modify
  • Portable to different machines

This was a major breakthrough in computing.

Why C Became So Popular

C gained popularity because it has a perfect balance between:

  • Low-level features (like memory access, pointers)
  • High-level features (like functions, loops)

So it is often called:

Middle-level programming language

Key Features of C

  • Fast and efficient
  • Portable (runs on different systems)
  • Structured programming support
  • Close to hardware (used in system programming)

Impact of C Language

C influenced many modern programming languages such as:

  • C++
  • Java
  • C#
  • JavaScript (syntax influenced)

Even today, C is used in:

  • Operating systems
  • Embedded systems
  • Compilers

Features of C Language

C is a powerful and flexible programming language that is known for three important characteristics:

It is a middle-level language, a structured language, and a programmer’s language.

1. C is a Middle-Level Language

C is called a middle-level language because it combines both low-level and high-level features.

  • Low-level features:
    • Direct memory access using pointers
    • Bitwise operations
    • Close interaction with hardware
  • High-level features:
    • Functions
    • Loops (for, while)
    • Conditional statements (if, switch)

This combination allows C to act as a bridge between hardware and software.

Because of this, C is used in:

  • Operating systems
  • Embedded systems
  • System software

Why It is Called Middle-Level

  • It is not purely low-level like assembly
  • It is not purely high-level like Python

It combines both:

Feature TypeCapability
Low-LevelMemory access, hardware interaction
High-LevelFunctions, loops, readability

Real-World Understanding

Think of C like a driver who can control both manual and automatic cars:

  • Low-level → Manual control (full power over system)
  • High-level → Automatic control (easy handling)

C gives you both control and simplicity

2. C is a Structured Language

C follows a structured programming approach, which means programs are divided into smaller parts.

  • Programs are organized using:
    • Functions
    • Loops
    • Conditions

Basic structure:

main() {
    // main logic
}

function1() {
    // task
}

1. main() Function

  • main() is the starting point of every C program
  • Execution of the program always begins from here

Think of it like the entry gate of the program

2. { } (Curly Braces)

  • These define a block of code
  • Everything inside { } is treated as one unit

This is the core idea of structured programming:

Divide program into blocks

3. // main logic

  • This represents the main operations of the program
  • It can include:
    • Function calls
    • Loops
    • Conditions

Instead of writing everything directly, we delegate tasks to other functions

4. function1()

  • This is a user-defined function
  • It performs a specific task

Example:

  • One function for login
  • One function for calculation
  • One function for display

This is called modular programming

5. Inside function1()

// task
  • Contains the actual logic for that specific task
  • Keeps code separate and organized

How Structure Works in Real Programs

A real structured C program looks like this:

#include<stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(2, 3);
    printf("%d", result);
    return 0;
}

readability, debugging, reusability, maintainability

Library

A library is a collection of predefined functions.

You don’t write them

You just use them by including the header file

Common C Libraries

1. stdio.h (Standard Input Output)

  • Used for input and output
  • Functions:
    • printf() → output
    • scanf() → input

2. stdlib.h (Standard Library)

  • General utilities
  • Functions:
    • malloc() → memory allocation
    • free() → deallocation
    • exit()

3. string.h

  • String operations
  • Functions:
    • strlen()
    • strcpy()
    • strcmp()

4. math.h

  • Mathematical operations
  • Functions:
    • sqrt()
    • pow()
    • sin()

5. conio.h (Not Standard)

  • Console input/output (mostly old compilers like Turbo C)
  • Functions:
    • getch()
    • clrscr()

What Linker Does

  • Takes your object file (hello.o)
  • Finds the actual printf code from the standard library
  • Combines them
